## IC1と基礎獲得式

頂点数が3の開始状態では、`y = floor(sqrt(頂点数)) = 1` となる。

このとき `(x / y)^y` は追加の多項式化としては適用せず、基礎獲得量 `x` をそのまま使用する。したがって、IC1の「基礎獲得式の `y` が10倍される」制約も、`y = 1` の間は効果を持たない。

頂点数が4以上になり、`y >= 2` となった時点から、多項式化およびIC1の制約を適用する。

## IC3の報酬

IC3の報酬は **ラップスピード ×1.1 のみ** とする。

`Generation` の実効スコア倍率に対する「IC3クリア後に指数を 2.00 から 2.10 へ変更する」記述は誤りであり、適用しない。

つまり、Generationの実効スコア倍率log10は、IC3のクリア状況によらず、次の計算を用いる。

```text
実効Generationスコア倍率log10 = Generationスコア倍率log10 x 2.00
IU 3-1購入後:
  実効Generationスコア倍率の指数を x1.5
実績3達成後:
  実効Generationスコア倍率 = 1 + (実効Generationスコア倍率 - 1) x 2
```
